◈ Dynamic Programming DP Concepts & Visualizations

Principle of Optimality

An optimal solution to a problem contains optimal solutions to its subproblems. If the global solution is optimal, every sub-path must also be optimal. This is the foundation of DP.

Overlapping Subproblems

The same subproblems are solved repeatedly. DP stores results in a table (memoization top-down or tabulation bottom-up) to avoid redundant computation.

DP vs Divide & Conquer

D&C: subproblems are independent (Merge Sort, Quick Sort). DP: subproblems overlap and share results. DP trades space for time efficiency.

⬡ Graph Algorithms Graph & Network Flow Visualizations
Breadth-First Search
Start Node
BFS explores nodes level by level using a queue. Applications: shortest path in unweighted graphs, web crawling, peer-to-peer networks, social network analysis.

Queue (FIFO)

BFS uses a queue. Enqueue the source, then neighbors. Process front → enqueue unvisited neighbors.

Level Order

All nodes at distance d are visited before nodes at distance d+1. Guarantees shortest path in unweighted graphs.

Complexity

Time: O(V + E). Space: O(V) for queue and visited array. V = vertices, E = edges.

Depth-First Search
Start Node
DFS explores as far as possible along each branch before backtracking. Applications: cycle detection, topological sort, maze solving, connected components.

Stack (LIFO)

DFS uses a stack (or recursion). Push start, pop and explore deepest unvisited neighbor first.

Discovery / Finish

Each node gets a discovery time and finish time, useful for topological sorting and SCC detection.

Complexity

Time: O(V + E). Space: O(V) for stack/recursion. Can detect back edges → cycles.

Topological Sort (Kahn's Algorithm)
Topological Sort orders vertices in a DAG so every edge u→v has u before v. Used in task scheduling, build systems, course prerequisites.
Strongly Connected Components (Kosaraju's)
SCC: maximal set of vertices mutually reachable from each other. Kosaraju's: 1) DFS finish-order, 2) transpose graph, 3) DFS in reverse finish-order.
Floyd-Warshall — All-Pairs Shortest Path
Floyd-Warshall computes shortest paths between ALL pairs of vertices. dp[i][j][k] = shortest path from i to j using vertices {0..k}. Time: O(V³).
Ford-Fulkerson — Max Flow / Min Cut
Max-Flow Min-Cut Theorem: maximum flow from source S to sink T equals minimum capacity of any S-T cut. Ford-Fulkerson finds augmenting paths until no more exist.

Augmenting Path

A path from S to T with available residual capacity. We push flow equal to the bottleneck (min edge capacity) along this path.

Residual Graph

After pushing flow f on edge u→v, add backward edge v→u with capacity f, allowing "undoing" of flow decisions.

Min-Cut

The set of edges whose removal disconnects S from T with minimum total capacity = max flow. A fundamental theorem in network flow.
